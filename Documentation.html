SupAgentic V2 Architecture

Welcome to the autonomous Masterpiece. SupAgentic V2 transforms the platform from a reactive CLI router into an Autonomous AI Operating System. The architecture consists of 5 interlocking pillars that solve tool amnesia, deployment overhead, and cross-agent interoperability.

The 5 Core Pillars

  • Compose Engine: YAML-driven Directed Acyclic Graph (DAG) for chaining agent scripts.
  • Phantom Context: Silent clipboard & terminal history injection for zero-shot context.
  • Unified Memory: Global SQLite FTS5 database acting as the swarm subconscious.
  • Forge Daemon: Autonomous GitHub scraping and self-patching router mechanism.
  • Ephemeral UIs: Dynamic HTML payload renderers over localhost HTTP bypasses.

Declarative Swarm Orchestration (Compose)

The Compose engine represents the critical bridge between fragmented Python, TypeScript, and Go tools. Rather than building API wrappers for all 171 tools, the Compose engine relies on standard I/O streams configured via a .yaml dependency matrix.

Phantom Context Engine

To establish fluid human-AI bridging, the Phantom Daemon constantly rotates an active local state capturing standard clipboard arrays and raw Windows PowerShell API histories into a secured phantom_cache.json.

At execution onset, tools receive PHANTOM_CONTEXT_CLIPBOARD and PHANTOM_CONTEXT_HISTORY directly injected into their shell ENV footprint.

Unified Memory Tissue

To resolve tool context amnesia, every active node within the Compose Matrix and independent CLI execution passes structural data into memory_bus.py—a frictionless SQLite layer heavily accelerated by FTS5 (Full Text Search).

Recursive Self-Expansion (Forge)

The system is inherently self-replicating. Upon deploying supagentic forge "tool idea", the V2 Router executes a programmatic multi-step loop:

  1. Triggers unauthenticated HTTPS GitHub queries sorting by stargazer count.
  2. Forks the optimal repository instantly to the user's local matrix.
  3. Taps the OpenAI APIs dynamically strictly for classification and parameter routing.
  4. Safely synthetically appends the registry dictionary to its own parent codebase supagentic.py.

Ephemeral Glass UIs

For data sets that scale beyond standard terminal bounds, V2 hooks outputs straight to supagentic_ui.py. This overrides raw CLI dumps with threaded web servers deploying 15-second lifespan HTML documents styled with cutting-edge CSS Glassmorphism logic.

$ python supagentic.py run openfishh --ui
