SupAgentic V2: Declarative Swarm Architecture

This document blueprints the "Compose" pillar of the SupAgentic V2 masterpiece vision. The goal is to move from running isolated tools to running declarative AI supply chains defined in .yaml manifests.

1. The Core Architecture

The Compose engine transforms SupAgentic from a simple CLI runner into a DAG (Directed Acyclic Graph) executor. It solves the critical bottleneck of interoperability by acting as the unified standard IO pipe between fundamentally diverse open-source scripts.

2. Pipeline Definition (YAML Chassis)

Instead of human DevOps routing the data, users define a declarative matrix. supagentic_compose.py handles tracking dependencies, extracting specific regex out of unstructured logs, and converting payloads.

version: '1.0'
name: "Competitive Intelligence Drip"

nodes:
  recon_node:
    tool: OpenFishh
    args: ["--target", "competitor.com", "--depth", "3"]
    extract:
      type: json
      keys: ["findings", "vulnerabilities"]

  report_gen:
    tool: claudeable
    depends_on: [recon_node]
    input_mapping: 
      # Feeds JSON extracted from OpenFishh to Claudeable as STDIN
      stdin: ${recon_node.extracted_data}
    args: ["--task", "Summarize findings into a sleek HTML dossier"]

  alert_node:
    tool: jacobo-workflows
    depends_on: [report_gen]
    input_mapping:
      env:
        WHATSAPP_PAYLOAD: ${report_gen.stdout}

3. The Inter-Process Communication (IPC) Moat

The challenge with piecing together 48+ isolated GitHub repos is that they don't share schemas. To make them compose gracefully, we build a Subprocess Isolation Wrapper.
